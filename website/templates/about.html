About

Problem: Is the sequence we created in the lab what we wanted?

The Sequence Validation Pipeline takes MiSeq sequencing results, compares them to a FASTA file of reference sequences, then generates and uploads an IGV and Excel file to ICE detailing the coverage and alignment scores of your MiSeq sequence.

Overview of Process

User input => Start Pipeline => SMB Server => FASTQ.GZ => BAM => VCF, BED, Covdepth, Call_Summary.txt => IGV.XML, HTML, Excel => End Pipeline => ICE

Input

  • Main Library Name (1): Name of Folder on SMB server where your MiSeq Run is stored
  • Sample IDs (1+): Names of MiSeq Sample IDs whose sequences you want validated
  • Reference Sequence (1): ICE Entry whose sequence we are validated Samples against
  • Email (1): Email will be notified when Sequence Validation Pipeline is complete and IGV/Excel files can be viewed in ICE
